# Вимоги до системи управління замовленнями ресторану

## Описання проєкту
Система управління замовленнями для ресторану, яка дозволяє:
- Реєструвати клієнтів
- Створювати замовлення з позиціями меню
- Обраховувати суму замовлення
- Відслідковувати статус замовлення
-처리оцінювати задоволення клієнта

## Функціональні вимоги

### 1. Управління клієнтами
- Система повинна підтримувати додавання нових клієнтів
- Кожен клієнт має унікальний ідентифікатор (ID)
- Кожен клієнт має ім'я та контактну інформацію

### 2. Управління меню
- Система повинна мати каталог доступних страв
- Кожна страва має назву, описання та вартість

### 3. Управління замовленнями
- Клієнт може створити замовлення
- Замовлення складається з позицій меню
- Система обраховує загальну суму замовлення
- Кожне замовлення має статус (NEW, PREPARING, READY, SERVED, CANCELLED)
- Можна скасувати замовлення

### 4. Обслуговування
- Персонал може змінювати статус замовлення
- Система видає сповіщення про зміни статусу

### 5. Оцінювання
- Клієнт може оцінити замовлення (1-5 зірок)
- Система зберігає оцінку

## Нефункціональні вимоги
- Програма компілюється та запускається з консолі
- Без зовнішніх баз даних та сервісів
- Мінімальна складність (прототип)
- Вся логіка виводиться у консоль
